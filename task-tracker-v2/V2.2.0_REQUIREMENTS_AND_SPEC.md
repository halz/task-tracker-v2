# タスク管理ツール開発 v2.2.0 要件定義・仕様書

作成日: 2026-03-13  
対象: Task Tracker v2.2.0

---

## 1. 目的

既存のタスク管理機能に「プロジェクト管理」と「メール連携強化」を追加し、
プロジェクト配下タスクをタイムライン（ガントチャート風）で直感的に管理できるようにする。

---

## 2. 要件定義

### 2.1 プロジェクト機能
- 既存タスクを親プロジェクトとしてプロジェクトページに登録できること（タスクのプロジェクト化）。
- 紐付け失敗や後付け対応のため、既存タスクを既存プロジェクトへ手動追加できること。
- プロジェクト化されたタスクは通常タスクリストから非表示にすること。

### 2.2 メール連携
- プロジェクト関連メールは、該当プロジェクトの配下タスク（プロジェクトタスク）として登録すること。
- 関連判定は手動・自動の両方をサポートすること。
  - 手動: タスクリストからプロジェクトを選択して追加。
  - 自動: 既存タスク化プロセスの判定ロジック（messageId / threadId / ticketId 等）を流用し、必要分を拡張。
- 重複登録は `messageId` ベースで防止すること。

### 2.3 ステータス管理
- 正式ステータス: `返信待ち / 対応中 / 完了 / 保留`。
- 通常タスクから除外するための追加ステータス: `プロジェクト化済み`。
- プロジェクトタスクにも正式ステータスを流用すること。
- メール受信・送信時の状態変化に追従すること。

### 2.4 ガントチャート風表示
- プロジェクトと直下タスクをタイムライン形式で表示すること。
- 各プロジェクト・プロジェクトタスクに締切日を設定できること。
- 時間粒度は `日 / 週 / 月` を切り替え可能にすること。
- 開始日未設定タスクは締切日のみ表示（マイルストーン表示）すること。

### 2.5 期限超過対応
- 期限超過タスクを色で強調（赤）すること。
- 警告バッジ/アイコンを表示すること。
- 優先度を自動で引き上げること。

### 2.6 レスポンシブ対応
- Desktop / Tablet / Mobile それぞれで可読性が担保される表示形式を提供すること。

---

## 3. 詳細仕様

### 3.1 用語定義
- **プロジェクト化**: 既存タスクを親プロジェクトへ昇格登録する操作。
- **プロジェクトタスク**: 特定プロジェクトに内包された子タスク。

### 3.2 非表示ルール
- 通常タスクリストは、`status IN ('完了', 'プロジェクト化済み')` を除外対象とする。
- 監査・デバッグ用に非表示含む表示トグルは任意で提供する。

### 3.3 メール関連付け仕様

#### 手動関連付け
1. ユーザーがタスク一覧で対象タスクを選択。
2. 「プロジェクトへ追加」操作を実行。
3. 既存プロジェクトを選択。
4. 選択プロジェクト配下にプロジェクトタスクとして登録。

#### 自動関連付け
1. メール受信時に既存ロジックで照合パラメータを抽出。
2. `messageId / threadId / ticketId` などで該当プロジェクト候補を照合。
3. 関連が確定した場合、配下タスクとして登録。
4. 照合不能時は未紐付けとして保留（手動補正可能）。

#### 重複防止
- `messageId` を一意キーとして扱う。
- 同一 `messageId` の再登録を拒否または upsert とする。

### 3.4 ステータス遷移（基準）
- メール受信で新規作成: `返信待ち`
- 作業開始: `対応中`
- 返信送信後の相手待ち: `返信待ち`
- ブロック時: `保留`
- 完了操作: `完了`

### 3.5 ガント表示仕様
- 既定粒度: `週`
- 切替: `日 / 週 / 月`
- 開始日あり: 期間バーで表示
- 開始日なし: 締切日のマイルストーン表示

### 3.6 締切超過仕様
- 期限超過は赤色強調
- 警告アイコン表示
- 優先度自動引き上げ（例: Medium→High）

### 3.7 画面別表示方針
- **Desktop**: フルガント（横軸タイムライン + 親子展開）
- **Tablet**: 簡易ガント（列圧縮）
- **Mobile**: 縦タイムライン中心（ガント風簡略表示）

---

## 4. データモデル（最小拡張案）

- `projects`
  - `id, name, deadline, status, originTaskId, createdAt, updatedAt`
- `tasks`
  - `id, title, status, deadline, startDate, projectId(nullable), sourceType, sourceRef, priority, createdAt, updatedAt`
- `emails`
  - `messageId(unique), threadId, ticketId, subject, from, receivedAt`
- `task_email_links`
  - `taskId, messageId, projectId`

---

## 5. API要件（概要）

- `POST /projects` プロジェクト作成（タスク昇格含む）
- `POST /projects/:id/tasks` 既存タスクをプロジェクトへ追加（手動）
- `POST /emails/sync` メール取り込み＆自動関連付け
- `PATCH /tasks/:id/status` ステータス更新
- `GET /projects/:id/timeline?granularity=day|week|month` ガント表示データ取得

---

## 6. 実装優先順位

1. ステータス定義追加（`プロジェクト化済み` 含む）
2. タスク→プロジェクト化機能
3. 既存プロジェクトへの手動追加
4. メール重複防止（messageId 一意）
5. 自動関連付けロジック拡張
6. ガント（日/週/月）表示
7. 期限超過の色・警告・優先度自動引き上げ
8. レスポンシブ最適化

---

## 7. 受け入れ条件（Acceptance Criteria）

- [ ] タスクをプロジェクト化できる
- [ ] プロジェクト化済みタスクが通常一覧に表示されない
- [ ] 紐付け失敗タスクを手動で既存プロジェクトへ追加できる
- [ ] メール重複登録が messageId で防止される
- [ ] プロジェクト配下タスクのステータスが正式状態で運用できる
- [ ] ガント表示で日/週/月切替ができる
- [ ] 開始日未設定タスクが締切マイルストーンで表示される
- [ ] 期限超過タスクに色・警告・優先度引き上げが反映される
- [ ] Desktop/Tablet/Mobile 各表示で可読性が確保される

---
